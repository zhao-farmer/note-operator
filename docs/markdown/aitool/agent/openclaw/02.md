# 二、配置openclaw

## 2.1 gateway的使用

1. 安装与启动

    ```
    # 安装Gateway后台服务
    openclaw gateway install

    # 启动Gateway（带基础向导）
    openclaw onboard --install-daemon

    # 手动启动Gateway（指定端口）
    openclaw gateway --port 18789 --verbose
    ```

2. 操作的基本命令

    ```sh
    openclaw gateway start # 启动
    openclaw gateway stop # 停车
    openclaw gateway restart # 重启
    ```
3. 如果一直打不开 `gateway`,在cmd中使用不通端口启动

    ```
    openclaw gateway --port 18780 --verbose
    ```

## 2.5 控台台命令

### **基础操作与服务控制**
这类命令是日常使用OpenClaw的起点和基础。

| 命令 | 功能说明 |
| :--- | :--- |
| `openclaw --version` | 查看版本号 |
| `openclaw --help` | 查看所有命令的帮助信息 |
| `openclaw onboard` | 启动交互式配置向导，新手初始化推荐使用 |
| `openclaw dashboard` | 打开Web控制台页面 |
| `openclaw gateway start` | 启动网关服务（核心后台服务） |
| `openclaw gateway status` | 查看网关运行状态 |
| `openclaw gateway restart` | 重启网关（修改配置后常用） |
| `openclaw status` | 查看OpenClaw整体运行状态 |

### **配置与管理**
OpenClaw的功能很强大，很多时候需要通过命令来精细化管理。

| 命令 | 功能说明 |
| :--- | :--- |
| `openclaw configure` | 重新进入配置向导 |
| `openclaw config get` | 查看当前配置 |
| `openclaw config set <键> <值>` | 设置配置项，例如 `openclaw config set models.default gpt-4` |
| `openclaw models list` | 列出已配置的AI模型 |
| `openclaw models set <模型名>` | 切换默认使用的AI模型 |
| `openclaw update` | 将OpenClaw更新到最新版本 |
| `openclaw logs` | 查看日志，用于排查错误 |

### **聊天中的斜杠命令**
这些命令是在Web界面、Telegram等聊天窗口中和AI对话时直接输入的，用于控制当前会话。

| 命令 | 功能说明 |
| :--- | :--- |
| `/new` | **新建会话**，清空上下文以节省Token |
| `/status` | 查看当前会话状态和Token消耗 |
| `/model` | 切换当前会话的AI模型 |
| `/compact` | 压缩上下文，在保留核心信息的同时节省Token |
| `/help` 或 `/commands` | 查看所有可用的斜杠命令 |

### 其他命令

| 命令 | 主要功能 | 使用场景 | 常用示例 |
| :--- | :--- | :--- | :--- |
| **`openclaw doctor`** | **全面健康检查**，诊断配置、端口、依赖等问题。 | 遇到奇怪的问题（如插件不工作、服务异常），首选此命令进行排查。 | `openclaw doctor` (仅检查)<br> `openclaw doctor --fix` (检查并尝试自动修复) |
| **`openclaw tui`** | **启动终端交互界面**，可以在命令行里直接和AI聊天。 | 适合在服务器维护、无法使用浏览器，或Web界面出问题时，通过终端与AI交互。 | `openclaw tui` (启动本地TUI)<br> `openclaw tui --url ws://<address>:<port>` (连接远程网关) |

简单来说，`doctor` 是“体检医生”，负责检查和修复系统问题；`tui` 是“后备聊天室”，让你在命令行里也能随时对话。
