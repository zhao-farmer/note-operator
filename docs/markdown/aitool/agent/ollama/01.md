#  一、Ollama认识与使用

## 1.1 Ollama是什么？

**Ollama** 是一个开源的、轻量级的本地大模型运行框架。它把运行AI模型的复杂性封装起来，让您像使用普通软件一样，通过简单的命令就能部署和运行Llama、DeepSeek、Qwen等主流大模型。

### **为什么选择Ollama？**

| 优势 | 说明 |
|:---|:---|
| **极简操作** | 一键安装，一行命令运行模型，无需手动配置Python环境、依赖库 |
| **数据隐私** | 所有计算都在本地完成，数据不会上传到云端，隐私安全有保障 |
| **免费使用** | 无API调用费用，不限次数，支持离线使用 |
| **生态丰富** | 支持数百种开源模型，可以随时切换 |
| **硬件加速** | 自动利用CUDA、Metal等GPU加速，充分发挥硬件性能 |
| **API支持** | 提供标准的REST API，方便集成到自己的应用中 |

### **与CUDA的关系**
您之前安装的CUDA为Ollama提供了"引擎"——Ollama会自动检测并使用CUDA来加速模型计算，让大模型的运行效率大幅提升。

## 1.2 Ollama的安装

### **各系统安装方式**

| 操作系统 | 安装方法 |
|:---|:---|
| **Windows** | 1. 访问 [ollama.com/download](https://ollama.com/download) <br> 2. 下载 `OllamaSetup.exe` <br> 3. 双击安装，完成后系统托盘会出现Ollama图标 |
| **macOS** | **方法一**：官网下载安装包<br>**方法二**：`brew install ollama` |
| **Linux** | 终端执行：`curl -fsSL https://ollama.com/install.sh \| sh` |

### **验证安装**

打开终端（或命令提示符），输入：

```bash
ollama --version
```
如果显示版本号（如 `ollama version 0.1.32`），说明安装成功。

## 1.3 使用Ollama运行模型

### **1. 运行第一个模型**

以阿里巴巴新发布的 **Qwen3（通义千问3）** 轻量版为例：

```bash
ollama run qwen3:1.7b
```

执行后发生了什么？

- **首次运行**：自动下载模型（约1.1GB）
- **加载模型**：进入对话界面，出现 `>>>` 提示符
- **开始对话**：直接输入问题即可

**试试这些：**

```
>>> 你好，请介绍一下自己
>>> 用Python写一个计算斐波那契数列的函数
>>> /bye  (退出对话)
```

### **2. 常用模型推荐**

| 模型 | 命令 | 特点 | 内存需求 |
|:---|:---|:---|:---|
| **通义千问3** | `ollama run qwen3:1.7b` | 中文能力强，轻量 | ~2GB |
| **通义千问3** | `ollama run qwen3:7b` | 中文能力强，性能更好 | ~6GB |
| **DeepSeek-Coder** | `ollama run deepseek-coder:6.7b` | 编程专用 | ~5GB |
| **Llama 3.2** | `ollama run llama3.2:3b` | Meta最新轻量版 | ~3GB |
| **Phi-3** | `ollama run phi3:mini` | 微软小模型，性能出色 | ~3GB |
| **Mistral** | `ollama run mistral:7b` | 通用能力强 | ~5GB |

### **3. Ollama核心命令大全**

| 命令 | 用途 | 示例 |
|:---|:---|:---|
| `ollama run <模型名>` | 运行模型（自动下载） | `ollama run qwen3:7b` |
| `ollama pull <模型名>` | 只下载，不运行 | `ollama pull deepseek-coder:6.7b` |
| `ollama list` | 查看已下载的模型 | `ollama list` |
| `ollama ps` | 查看正在运行的模型 | `ollama ps` |
| `ollama rm <模型名>` | 删除模型 | `ollama rm qwen3:1.7b` |
| `ollama stop <模型名>` | 停止运行中的模型 | `ollama stop qwen3:7b` |
| `ollama cp <源> <目标>` | 复制模型 | `ollama cp qwen3:7b my-qwen` |
| `ollama pull <模型名>` | 更新模型 | `ollama pull qwen3:7b` |

## 1.4 Ollama的高级使用

#### **1. 通过API调用模型**

Ollama安装后，会自动在后台启动一个API服务（默认地址：`http://localhost:11434`）。您可以通过HTTP请求来调用模型。

**Python示例：**
```python
import requests
import json

# 普通对话
response = requests.post(
    'http://localhost:11434/api/generate',
    json={
        "model": "qwen3:1.7b",
        "prompt": "解释一下什么是人工智能",
        "stream": False
    }
)
print(response.json()['response'])

# 流式输出（实时显示）
response = requests.post(
    'http://localhost:11434/api/generate',
    json={
        "model": "qwen3:1.7b",
        "prompt": "写一首关于春天的诗",
        "stream": True
    },
    stream=True
)
for line in response.iter_lines():
    if line:
        print(json.loads(line)['response'], end='', flush=True)
```

### **2. 创建自定义模型**

您可以通过Modelfile来创建定制化的模型版本。

**创建步骤：**

1. 创建一个文件 `Modelfile`：

    ```dockerfile
    FROM qwen3:1.7b

    # 设置系统提示词
    SYSTEM "你是一个专业的Python编程助手，专注于提供简洁高效的代码示例。"

    # 设置参数
    PARAMETER temperature 0.7
    PARAMETER top_p 0.9
    ```

2. 构建自定义模型：

    ```bash
    ollama create my-python-assistant -f ./Modelfile
    ```

3. 运行自定义模型：

    ```bash
    ollama run my-python-assistant
    ```

### **3. 配置Ollama服务**

Ollama支持多种环境变量配置：

**Windows (设置系统环境变量)：**
- `OLLAMA_HOST=0.0.0.0:11434` # 监听所有网络接口
- `OLLAMA_MODELS=D:\ollama_models` # 模型存储位置
- `OLLAMA_NUM_PARALLEL=4` # 并行处理请求数
- `OLLAMA_KEEP_ALIVE=5m` # 模型保持加载时间

**Linux/macOS：**
```bash
export OLLAMA_HOST=0.0.0.0:11434
export OLLAMA_MODELS=/data/ollama/models
ollama serve
```

## 1.5 重要注意事项

### **1. 安全警告**
**非常重要！** 2025年3月，国家网络安全通报中心曾发文指出Ollama默认配置存在未授权访问风险。请务必遵守：

- **只在本机使用**：默认配置 `127.0.0.1:11434` 是安全的
- **不要直接暴露公网**：不要将11434端口直接映射到公网
- **如需远程访问**：必须配置防火墙规则、使用VPN或安全代理

### **2. 硬件要求**

| 模型大小 | 推荐内存 | 适用场景 |
|:---|:---|:---|
| 1B-3B模型 | 8GB+ | 轻量对话、文本分类 |
| 7B模型 | 16GB+ | 通用对话、代码生成 |
| 13B模型 | 32GB+ | 复杂推理、专业任务 |
| 30B+模型 | 64GB+ | 科研、深度分析 |

### **3. 磁盘空间管理**
- 查看模型大小：`ollama list` 会显示每个模型的大小
- 清理无用模型：`ollama rm <模型名>`
- 修改存储位置：通过 `OLLAMA_MODELS` 环境变量指定

## 1.7 常见问题解决

| 问题 | 可能原因 | 解决方法 |
|:---|:---|:---|
| 模型下载慢 | 网络问题 | 使用代理，或手动下载后导入 |
| CUDA不可用 | 驱动问题 | 运行 `nvidia-smi` 检查驱动 |
| 内存不足 | 模型太大 | 选择更小的模型或量化版本 |
| 响应慢 | 硬件性能 | 减少并发请求，调整参数 |
| API连接失败 | 服务未启动 | 运行 `ollama serve` 启动服务 |
