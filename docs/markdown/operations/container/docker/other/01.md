# docker日常记录

## 1. docker的映射顺序

1. **端口映射 `-p`或 `--publish`**

    ```bash
    # 语法: -p <宿主机端口>:<容器端口>
    # 含义: 将宿主机的端口映射到容器的端口

    # 示例1: 将宿主机的80端口映射到容器的80端口
    -p 80:80

    # 示例2: 将宿主机的8080端口映射到容器的80端口
    -p 8080:80

    # 示例3: 将宿主机的随机端口映射到容器的80端口
    -p 80  # 自动分配宿主机端口
    ```

2. **磁盘映射 `-v`或 `--volume`**

    ```bash
    # 语法: -v <宿主机路径>:<容器路径>[:选项]
    # 含义: 将宿主机的目录/文件映射到容器的目录/文件

    # 示例1: 将宿主机的/home/data映射到容器的/data
    -v /home/data:/data

    # 示例2: 将宿主机的目录映射到容器，只读权限
    -v /home/data:/data:ro

    # 示例3: 使用SELinux标签
    -v /home/data:/data:z
    ```


3. **Docker 映射的顺序永远是：**

    1. **端口映射**：`宿主机端口:容器端口`
        
    2. **磁盘映射**：`宿主机路径:容器路径`
