一、 初识openclaw ​

1.1 openclaw介绍 ​

  1. openclaw 是什么 OpenClaw（曾用名：Clawdbot、Moltbot）是一个近期非常火爆的开源个人AI智能体（Agent）平台。

    简单来说，它不仅仅是一个能陪你聊天的AI对话框，而是一个能24小时运行在你自己的设备上，帮你自动执行各种任务的“数字管家”或“数字分身”。

  2. 以下是它的核心特点：

    • 本地部署与隐私安全：

      • OpenClaw 设计为运行在用户自己的硬件上（如个人电脑、Mac Mini、NAS或轻量云服务器），而不是完全依赖云端服务。这意味着你的数据和操作记录主要掌握在自己手中，隐私性更强。

    • “特工”而非“顾问”：

      • 传统的AI通常是“你问它答”的顾问模式。
      • OpenClaw 则是“特工”模式，它可以主动执行任务。例如：自动整理邮箱、管理日程、浏览网页获取信息、执行脚本、处理文件、跨应用协同工作等。

    • 多平台交互：

      • 它可以通过你常用的聊天软件进行控制，支持 WhatsApp、Telegram、Discord 等接口。你可以像给朋友发微信一样给它发送指令，它就会在后台默默完成任务。

    • 模块化架构：

      • 其核心架构通常包含网关（Gateway）、智能体（Agent）、技能（Skills）和记忆（Memory）模块。这种设计让它能通过安装不同的“技能”来无限扩展功能，并且拥有长期记忆能力，能记住你的偏好和历史上下文。

    • 开源与社区生态：

      • 作为一个开源项目（主要在GitHub上），它拥有活跃的开发者社区。由于其实用性强，曾在短时间内获得极高的关注度（数万Star）。
      • 注：该项目曾因名称与Anthropic公司的Claude过于相似而收到关切，因此从Clawdbot/Moltbot更名为OpenClaw。

  3. 总结：

    如果你想拥有一个私有化、能真正干活、全天候待命的AI助手，而不是仅仅用来闲聊，OpenClaw 就是为此目的而设计的工具。目前阿里云等平台也提供了一键部署方案，降低了普通用户的使用门槛。

1.2 下载openclaw ​

  1. 文档与官网

    • openclaw 官网: https://openclaw.ai/
    • openclaw github地址：https://github.com/openclaw/openclaw
    • openclaw 文档地址: https://docs.openclaw.ai/zh-CN

  2. 安装程序

    1. 准备工作

      • 系统要求：Windows 10/11 (推荐通过WSL2)、macOS 12+ 或 Linux (如Ubuntu 22.04+)。
      • 核心依赖：Node.js 版本需 ≥ 22.0.0。
      • 必备凭证：一个大语言模型的 API Key，例如从阿里云百炼、DeepSeek、OpenAI或Anthropic获取。

    2. 安装Node.js：

      • Windows用户：从 Node.js官网 下载并安装最新的LTS版本，安装时记得勾选“Add to PATH”。
      • macOS：终端执行 brew install node@22
      • Linux用户:

        • Ubuntu / Debian (使用 Nodesource 源)

          sh
          # 安装 setup 脚本
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
# 安装 Node.js
sudo apt-get install -y nodejs

        • CentOS / RHEL / rolky / Fedora

          sh
          # 启用 Nodesource 源
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
# 安装 node.js
sudo yum install -y nodejs
sudo dnf install -y nodejs

    3. 安装openclaw

      • npm安装

        sh
        npm i -g openclaw

      • 脚本安装

        sh
        # macOS / Linux
curl -fsSL https://openclaw.ai/install.sh | bash
# Windows (PowerShell)
iwr -useb https://openclaw.ai/install.ps1 | iex

1.3.3 配置项的解释 ​

根据多个官方和社区文档，这个向导会包含以下所有可能的配置项（实际出现的顺序和选项会根据你的选择动态变化）：

  1. 风险确认（Security Notice）

    • 提示内容：确认你了解OpenClaw拥有强大权限，存在固有风险
    • 选项：Yes, I understand this is powerful and inherently risky.
    • 说明：必须选择同意才能继续安装

  2. 引导模式（Onboarding Mode）

    • 提示内容：选择配置向导的模式
    • 选项：
      • QuickStart - 快速启动，推荐大多数用户使用
      • Custom - 自定义配置，适合高级用户
    • 说明：QuickStart会提供合理的默认值，减少交互步骤

  3. 网关运行位置（Where will the Gateway run?）

    • 提示内容：选择OpenClaw网关的运行位置
    • 选项：
      • Local (this machine) - 本地运行
      • Remote - 远程运行
    • 说明：一般选择本地

  4. 配置部分选择（Select sections to configure）

    • 提示内容：选择你想要配置的部分
    • 选项：
      • Model - 配置模型
      • Channels - 配置通信渠道
      • Skills - 配置技能
      • Hooks - 配置钩子
    • 说明：可以多选，也可以单独配置某一项

  5. 模型/认证提供商（Model/auth provider）

    • 提示内容：选择你想要使用的大模型提供商
    • 选项（非常丰富，取决于你的需求）：
      • Skip for now - 跳过，稍后配置
      • OpenAI - OpenAI系列模型
      • Anthropic - Claude系列模型
      • MiniMax - MiniMax模型（支持OAuth或API Key）
      • Qwen - 通义千问（支持免费OAuth）
      • LongCat - LongCat系列模型
      • Google - Google Gemini模型
      • DeepSeek - 深度求索模型
      • 自定义 - 手动输入API地址

  6. 认证方式（Auth method）

    • 提示内容：选择该提供商的认证方式
    • 选项（取决于所选提供商）：
      • API Key - 输入API密钥
      • OAuth - 通过网页授权登录
      • Skip - 跳过

  7. API密钥输入（API Key）

    • 提示内容：粘贴你的API密钥
    • 说明：如果选择了API Key方式，会提示你输入

  8. 端点选择（Endpoint）

    • 提示内容：选择服务端点
    • 选项（以MiniMax为例）：
      • CN - 国内端点
      • Global - 海外端点
    • 说明：根据你的网络环境和账号类型选择

  9. 模型过滤（Filter models by provider）

    • 提示内容：是否按提供商过滤模型
    • 选项：
      • All providers - 显示所有提供商
      • Selected provider only - 仅显示当前选择的提供商
    • 说明：影响后续模型选择列表

  10. 默认模型选择（Default model）

    • 提示内容：从列表中选择默认使用的模型
    • 选项：根据你选择的提供商，显示可用模型列表
    • 说明：可以选择Keep current保留当前设置

  11. 通信渠道配置（Channels）

    • 提示内容：选择你想要连接的通信渠道
    • 选项（可多选，或选择Skip）：
      • Skip for now - 跳过，稍后配置
      • Telegram - 电报
      • WhatsApp - 瓦次艾普
      • Slack - 斯拉克
      • Discord - 迪斯科
      • 飞书 - 飞书（中文版特有）
      • 微信 - 微信（开发中）
      • Signal - 信号
      • iMessage - 苹果即时通讯
      • Microsoft Teams - 微软团队
      • Matrix - 矩阵
      • Zalo - 扎洛
    • 说明：选择后，会进一步引导你配置对应渠道的Bot Token等信息

  12. 技能配置（Configure Skills）

    • 提示内容：是否启用本地工具技能
    • 选项：
      • Yes - 启用（推荐）
      • No - 不启用
    • 说明：技能是OpenClaw的核心能力，如浏览器控制、文件操作等

  13. 技能包管理器选择（Preferred node manager for skill installs）

    • 提示内容：选择用于安装技能依赖的包管理器
    • 选项：
      • npm - NPM包管理器
      • pnpm - PNPM包管理器
      • yarn - Yarn包管理器
      • Skip - 跳过

  14. 安装缺失技能依赖（Install missing skill dependencies）

    • 提示内容：是否自动安装缺失的技能依赖包
    • 选项：
      • Yes - 安装
      • Skip for now - 跳过，稍后手动安装
      • No - 不安装
    • 说明：选择Yes可以确保技能开箱即用

  15. 技能API密钥配置（Skill API Keys）

    • 提示内容：为各个技能配置所需的API密钥
    • 说明：如果你安装了需要额外API密钥的技能（如联网搜索技能tavily-search），这里会逐一询问你是否配置

  16. 钩子配置（Hooks）

    • 提示内容：是否启用钩子功能
    • 选项：
      • Skip for now - 跳过
      • 具体钩子列表（如session-memory、command-logger等）
    • 说明：钩子用于扩展OpenClaw的功能，如自动保存会话、记录命令日志

  17. 网关服务安装（Gateway Service）

    • 提示内容：是否安装网关服务（开机自启）
    • 选项：
      • Install - 安装
      • Restart - 重装或更新或出现下面这个，不要选
      • Reinstall - 重新安装
      • Skip - 跳过
    • 说明：选择Install会将OpenClaw安装为系统服务，开机自动运行

  18. 启动方式选择（How to Hatch Your Bot）

    • 提示内容：选择如何启动和体验你的AI助手
    • 选项：
      • Hatch in TUI - 终端界面模式（推荐）
      • Open Dashboard - 打开Web控制台
      • Do this later - 稍后手动启动
    • 说明：TUI模式可以直接在终端对话测试，Web模式会打开浏览器界面

1.3.5 实用建议 ​

  1. 初次安装选QuickStart：如果想快速上手，全程选QuickStart + Skip for now，先让服务跑起来，之后再慢慢配置其他功能

  2. 模型选择：

    • 有API Key：选你常用的模型（OpenAI/Anthropic/DeepSeek等）
    • 没有API Key：选Qwen OAuth（免费）

  3. 渠道配置可后补：Telegram/WhatsApp等渠道配置比较复杂，建议先Skip，跑通后再用openclaw channels add命令添加

  4. 技能推荐启用：建议选Yes安装技能，这是OpenClaw的核心价值所在

  5. 配置修改：

    • 重新运行向导：openclaw configure
    • 直接清理目录（重新配置）：rm -rf ~/.openclaw
    • 直接编辑配置文件：~/.openclaw/openclaw.json
    • Web界面修改：访问 http://127.0.0.1:18789 进入Config页面

  6. 在UI页面，发送信息进行验证

1.3.6 安装出现的问题 ​

  1. 权限不足问题

    • Q：Tip: rerun from an elevated PowerShell (Start → right-click → Run as administrator) or skip service install.
    • A: 使用管理员打开PowerShell

  2. API问题

    • Q: 如果使用qianwen模型，直接登录了
    • A: 目前配置qianwen大模型是直接登录，不过好处是不收费

  3. 启动问题

    • Q: 启动后，打开 http://127.0.0.1:18789/ 出现错误提示

    • A: 查看是不是缺少了这两步

      1. 网关要启动

      2. 必须要浏览器输入这个带token的url进行认证

  4. 全局更新(重新安装)

sh
npm i -g openclaw@latest
